# NvDsInferParseCustomResnet 底层逻辑详解

## 一、这个文件是干什么的？（一句话概括）

这是 **NVIDIA DeepStream** 框架的一个**自定义模型输出解析器**。神经网络做完推理后输出的只是一堆"原始浮点数"，这个文件把它们翻译成 DeepStream 能理解的"检测框"（坐标 + 置信度）。

---

## 二、AI 检测网络必备基础知识

> 在看懂代码之前，先搞清楚几个底层概念。你不需要理解它们背后的微积分推导，只需要知道"它在干什么"。

### 2.1 卷积（Convolution）到底在干什么？

  **3×3 的放大镜**（叫"卷积核"），在图片上从左到右、从上到下一格一格地滑动。每到一格，放大镜里的 9 个像素各乘一个权重再求和，得到一个输出值。滑完整张图，你就得到一张新的"特征图"。

```
输入图片 (比如 480×270)          卷积核 (3×3 权重矩阵)        输出特征图
┌─────────────────┐              ┌───┬───┬───┐            ┌─────────────┐
│ 128 130 129 ... │              │ 1 │ 0 │-1 │            │ 边缘强度     │
│ 127 255 128 ... │  ──×──▶      │ 2 │ 0 │-2 │  ──▶      │ 高 低 高 ... │
│ 125 124 130 ... │              │ 1 │ 0 │-1 │            │ 低 高 低 ... │
│ ...             │              └───┴───┴───┘            └─────────────┘
└─────────────────┘
                                        ↑
                                   这个卷积核在检测"垂直边缘"
                                   (左边×正数 + 右边×负数 = 有边缘就输出大值)
```

**一个卷积核 = 一种特征检测器。** 有的检测水平边缘，有的检测纹理，有的检测颜色过渡。网络越深，检测的特征越抽象（第一层检测边角 → 中间层检测形状 → 深层检测"像车牌的东西"）。

### 2.2 通道（Channel）是什么意思？

一张普通彩色图片有 **3 个通道**（R、G、B）。经过卷积后，通道数会变：

```
输入: 3 通道 (RGB)
  ↓ 64 个卷积核
输出: 64 通道 ← 每一层通道 = 一种"特征视角"
  ↓ 128 个卷积核
输出: 128 通道
```

**通道 ≈ 从不同角度描述同一个位置的特征。** 比如某一层 64 通道的某个位置，可能同时记录了"这里有边缘""这里有红色""这里很暗"等信息。

### 2.3 张量（Tensor）—— 数据的"形状"

你看到的 `[16, 17, 30]` 这种东西叫**张量的形状**：

```
[16, 17, 30]
  │   │   │
  │   │   └── 宽度: 30 列 (水平方向)
  │   └────── 高度: 17 行 (垂直方向)
  └────────── 通道数: 16 层 (每个位置有 16 个数值)
```

可以想象成 **16 张 17×30 的表格叠在一起**。代码里的 `output_bbox_buf[索引]` 就是在这一叠表格里按位置取一个 float。

### 2.4 Stride（步长）和 下采样 —— 图片为什么越变越小？

卷积每次移动的格子数叫 **stride**。

```
stride=1:  每次滑动 1 格 → 输出图大小 ≈ 输入图大小
stride=2:  每次滑动 2 格 → 输出图宽高减半！

输入 480×270，stride=2:
  ┌──┬──┬──┬──┬──┬──┐
  │x │  │x │  │x │  │    每两个取一个
  └──┴──┴──┴──┴──┴──┘    → 输出 240×135
```

**为什么要缩小？**
- 计算量：图片越小，卷积越快（4 次减半后计算量 ×256 倍减少）
- 视野：stride 后每个输出像素"看到"的原始范围变大了（感受野增大），更容易检测大物体

**这个模型里：** 输入 480×270，经过 4 次 stride=2，变成 30×17 的网格。**反过来想，** 30×17 的网格上每个点，对应原图 16×16 的一片区域——这个点"负责"检测这片区域里有没有车牌。

### 2.5 ResNet 和 Shortcut（跳连接）—— 为什么网络能堆很深？

**问题：** 普通的卷积网络堆到 20 层以上就训练不动了——梯度消失，前面层的权重几乎不更新。

**ResNet 的解法：加一条"高速公路"绕过卷积层。**

```
普通网络:                    ResNet:
  input                       input
    │                            │
    ├── conv                    ├── conv
    │                            │   │
    ├── conv                    ├── conv
    │                            │   │
    └── output                   └─(+)── output
                                   ↑
                               shortcut (输入原样跳过来)
```

**直觉理解：**
- 普通网络每一层必须学"新的东西"，前面的信息容易丢失
- ResNet 每一层只需要学"和输入的**差异**"（residual = 残差），输入信息通过 shortcut 直接保住

这个模型的名字 `NvDsInferParseCustomResnet` 里的 **Resnet** 就来自这里。

在 [fd_lpd.prototxt](fd_lpd_model/fd_lpd.prototxt) 里你可以看到这个模式反复出现：

```prototxt
# Block 1a 的三段式：
block_1a_conv_1      → 3×3 卷积 (主路径第一步)
block_1a_conv_shortcut → 1×1 卷积 (shortcut，只调整通道数对齐)
block_1a_conv_2      → 3×3 卷积 (主路径第二步)
add_1                → 主路径 + shortcut ← 残差连接的核心：逐元素相加
```

### 2.6 Sigmoid 和 ReLU —— 激活函数是干嘛的？

神经网络里的卷积和全连接层都是**线性运算**（加权求和）。线性套线性还是线性——如果没有激活函数，100 层的网络等效于 1 层。

激活函数引入**非线性**，让网络有能力拟合任意复杂的关系。

**ReLU**（Rectified Linear Unit）：
```
ReLU(x) = max(0, x)

   ↑
 3 │        ●──●──●    负数全变0，正数保持不变
 2 │      ●
 1 │    ●
 0 │● ● ● ─ ─ ─ ─ ─
───┼────────────────→
   │
```
简单高效，用在网络的**中间层**。但输出范围是 0~+∞，不适合表达"概率"。

**Sigmoid**：
```
Sigmoid(x) = 1 / (1 + e^(-x))

   ↑
1.0│      ～～～ ●●●～～～     把任意实数压缩到 0~1
0.5│        ／
   │      ●
0.0│～～～┘
───┼────────────────→
```
把任意实数压到 0~1，天然适合表达"概率"/"置信度"。用在这个模型的 **cov 输出层**。

### 2.7 FP16（半精度）vs FP32 —— 为什么 build_engine.sh 要用 `--fp16`？

计算机存小数用 **浮点数**：
- **FP32**：32 位存一个数，精度高，但占内存大，算得慢
- **FP16**：16 位存一个数，精度低一点，但占内存减半，速度快约 2 倍

```
一个 500MB 的 FP32 模型 → FP16 后 ≈ 250MB
对于 Jetson（嵌入式设备，内存有限），FP16 几乎是必须的
```

精度损失对于**检测任务**来说通常可以忽略（检测框差 0.5 个像素肉眼看不出来）。但如果做科学计算（比如天气预报模拟），就不能用 FP16。

## 三、目标检测的核心范式：网格 + bbox + cov

### 3.1 检测 vs 分类 —— 有什么不同？

| 任务 | 输入 | 输出 | 难度 |
|------|------|------|------|
| **图像分类** | 一张图 | "这是猫" | 简单 |
| **目标检测** | 一张图 | "左上角有只猫，右下角有个人，分别在哪" | 复杂 |

检测 = **同时回答两个问题**：这是什么？（分类） + 在哪里？（定位）

### 3.2 网格化检测（Grid-based Detection）的整体思路

神经网络不是直接输出"车牌在 (100,50) 到 (200,150)"，而是：

```
第一步：把图片分成 N×M 个网格格子。

    图片 480×270 → 分成 30×17 个格子，每格 16×16 像素

    ┌──┬──┬──┬──┬──────────┬──┐
    │  │  │  │  │          │  │
    ├──┼──┼──┼──┼──────────┼──┤
    │  │  │🚗│🚗│   ← 格子(3, 2)  │
    ├──┼──┼──┼──┼──────────┼──┤
    │  │  │🚗│🚗│   ← 格子(3, 3)  │
    ├──┼──┼──┼──┼──────────┼──┤
    │  │  │  │  │          │  │
    └──┴──┴──┴──┴──────────┴──┘

第二步：每个格子独立回答两个问题——
    问题 A: "我这里面有没有车牌？"        → 输出 cov (置信度)
    问题 B: "如果有，框的精确位置是多少？" → 输出 bbox (坐标偏移量)

第三步：把所有格子里"有东西"的框汇总，就是检测结果。
```

### 3.3 cov 层：置信度 —— "这格子里有东西吗？"

```
cov = confidence（置信度）

张量形状: [类别数, 网格高, 网格宽] = [4, 17, 30]
           ↑
    每个类别独立判断一次

cov[classId][row][col] = 0.0 ~ 1.0 的浮点数
```

**为什么要经过 Sigmoid？**

网络内部的卷积输出可能是 -100 到 +100 的任何值。Sigmoid 把它变成 0~1，就可以当作概率来用：

```
卷积原始输出: 3.2   →   Sigmoid(3.2) ≈ 0.96  → "96% 把握这里有车牌"
卷积原始输出: -0.4  →   Sigmoid(-0.4) ≈ 0.40  → "40% 把握，不太确定"
卷积原始输出: -5.1  →   Sigmoid(-5.1) ≈ 0.006 → "基本确定没有"
```

代码里的体现：
```cpp
const float threshold = detectionParams.perClassPreclusterThreshold[c];
// 比如 threshold = 0.5
// cov = 0.96 → 大于 0.5 → 保留
// cov = 0.40 → 小于 0.5 → 丢弃
if (confidence < threshold) continue;
```

### 3.4 bbox 层：边界框坐标 —— "框到底在哪？"

```
bbox = bounding box（边界框）

张量形状: [类别数 × 4, 网格高, 网格宽] = [16, 17, 30]
           ↑
    每个类别 4 个值: x1, y1, x2, y2
```

**为什么是 4 个值？**

一个矩形框只需要两个对角点就能确定：

```
    x1,y1 ●─────────┐
          │  车牌   │
          └─────────● x2,y2

x1 = 左上角 X 坐标    y1 = 左上角 Y 坐标
x2 = 右下角 X 坐标    y2 = 右下角 Y 坐标
```

**为什么存储的是"偏移量"而不是直接存坐标？**

直接让网络输出 0~480 的大数值，训练时梯度会爆炸。所以做法是：

1. 网络只输出一个**小范围的偏移量**（约 -1 到 +1）
2. 偏移量是**相对于网格中心**的
3. 解码时用网格中心 + 偏移量 = 真实坐标

```
示意图（一个网格内）：

    网格左上角                       网格右下角
    (0, 0)                         (16, 16)
      ┌──────────────────────────────┐
      │                              │
      │       gc_center_x            │
      │       (8, 8)                 │
      │         ●                    │
      │    ┌─────────┐               │
      │    │  offset  │              │
      │    │  (x1,y1) │              │
      │    │    │     │              │
      │    │    └─────┤─── offset    │
      │    │  检测框  │   (x2,y2)    │
      │    └─────────┘               │
      │                              │
      └──────────────────────────────┘
```

### 3.5 bbox 和 cov 的关系 —— 一对组合拳

```
每个网格位置 (row, col) 同时输出:

  cov[class][row][col] = 0.87   → "位置 (row,col) 有 87% 把握有一个 class 类物体"
  bbox[class][row][col] = {
      x1_offset, y1_offset,     → "如果有，框的左上角相对网格中心偏这么多"
      x2_offset, y2_offset      → "如果有，框的右下角相对网格中心偏这么多"
  }

解析时:
  1. 先看 cov → 低于阈值直接跳过，不浪费时间去解码 bbox
  2. cov 够高 → 解码 bbox 得到真实坐标
  3. 裁剪到图片范围内 → 输出
```

## 四、坐标解码的完整数学推导

### 4.1 下采样倍数

```
输入图片: 270 × 480 (高度 × 宽度)
网格大小:  17 × 30

每个格子的像素跨度:
  stride_y = ceil(270 / 17) = 16 像素
  stride_x = ceil(480 / 30) = 16 像素
```

模型内部经过了 **4 次 stride=2 的下采样**（2⁴ = ×16 倍缩小），270÷16≈17, 480÷16=30。

这意味着：**最后一层特征图上的 1 个像素，对应原图 16×16 的区域。**

### 4.2 网格中心点计算

```cpp
// stride = 16, bbox_norm = 35.0

// 第 w 列的网格中心 X 坐标（归一化）
gc_centers_x[w] = (w * 16 + 0.5) / 35.0

// 第 h 行的网格中心 Y 坐标（归一化）
gc_centers_y[h] = (h * 16 + 0.5) / 35.0
//                   │      │     │
//                   │      │     └── 归一化系数（训练时定死的）
//                   │      └──────── 0.5 =这是 NVIDIA DetectNet 训练时的设计选择。这个模型的参考系是"格子左上角 + 微小偏移"，而不是格子中心
//                   └─────────────── 格子索引 × 像素跨度 = 格子左上角的像素位置
```

**`bbox_norm = 35.0` 到底是什么？**

> 35.0 是 **NVIDIA DetectNet 框架的标准归一化常数**，来自 NVIDIA 2018 年的原始训练代码。这个值把像素坐标缩放到一个神经网络更容易处理的范围（大约 0~15 而不是 0~480）。**它不是 Codex 编的——是 NVIDIA 官方模型的固定参数。** 如果你换了自己训练的模型，必须确认训练脚本里的这个值是多少。

### 4.3 解码公式

```cpp
// 左上角: 网格中心 减去 偏移量
rect_x1 = (output_x1 - gc_center_x) × (-35.0)
        = (gc_center_x - output_x1) × 35.0

// 右下角: 网格中心 加上 偏移量
rect_x2 = (output_x2 + gc_center_x) × 35.0
```

**直观理解**：网络输出的 `output_x1` 是一个归一化的"左上角位置"，解码就是反归一化。

### 4.4 图解一个具体例子

```
归一化空间 (所有值除以 35.0):

  网格(0)          网格(1)          网格(2)
  [0───────16]    [16───────32]    [32───────48]     ← 像素空间 (每格 16px)
     │                │                │
     ↓                ↓                ↓
  gc ≈ 0.23        gc ≈ 0.69        gc ≈ 1.14        ← 除以 35.0 后
  (8/35)           (24/35)          (40/35)

假设网格(1)的模型输出:
  output_x1 = 0.5    (模型说: 框的左边界在归一化空间的 0.5 处)
  output_x2 = 0.9    (模型说: 框的右边界在归一化空间的 0.9 处)

解码回像素:
  rect_x1 = (0.69 - 0.5) × 35.0 = 6.65  → 第 7 像素
  rect_x2 = (0.69 + 0.9) × 35.0 = 55.65 → 第 56 像素

→ 车牌水平范围: 7 ~ 56 像素 (在原图上约 50 像素宽，合理)
```

## 五、完整的模型真相

### 5.1 模型来源

从 [fd_lpd.prototxt](fd_lpd_model/fd_lpd.prototxt) 可以看出，这是 **NVIDIA 官方 2018 年的 "FD_LPD_Redactor" 模型**（Caffe 格式），不是用户自己训练的。

FD = Face Detection（人脸检测），LPD = License Plate Detection（车牌检测）。

### 5.2 网络结构：双分支共享输入

```
                        输入图片
                     [3 × 270 × 480]
                           │
            ┌──────────────┴──────────────┐
            │                             │
     Branch 1 (LPD 车牌检测)        Branch 2 (FD 人脸检测)
     主干: ResNet-10 (大)           主干: ResNet-10 (小)
            │                             │
     Conv1: 7×7, 64ch, s2          Conv1: 7×7, 48ch, s2
     Block1: 64→64, s2             Block1: 48→64, s2
     Block2: 64→128, s2            Block2: 64→128, s2
     Block3: 128→256, s2           Block3: 128→160→128, s2
     Block4: 256→512, s1           Block4: 128→64→88, s1
            │                             │
   特征图: 512×17×30              特征图: 88×17×30
            │                             │
     ┌──────┴──────┐               ┌──────┴──────┐
     │              │               │              │
  bbox头          cov头           bbox头          cov头
  1×1 conv      1×1 conv         1×1 conv       1×1 conv
  12通道         3通道            4通道           1通道
  无激活          +Sigmoid        无激活           +Sigmoid
     │              │               │              │
     └──────┬───────┘               └──────┬───────┘
            │                              │
     conv2d_bbox_branch_1       output_bbox_branch_2
     conv2d_cov/Sigmoid_branch_1 output_cov/Sigmoid_branch_2
            │                              │
            └──────────┬───────────────────┘
                       │  Concat 拼接
            ┌──────────┴──────────┐
            │                     │
      output_bbox             output_cov
      [16 × 17 × 30]          [4 × 17 × 30]
      通道0-3: 人脸bbox       通道0: 人脸置信度
      通道4-15: 车牌bbox×3    通道1-3: 车牌×3
```

**关于 1×1 卷积（检测头）：**

你可能奇怪——1×1 卷积有什么意义？它不是只看 1 个像素吗？

1×1 卷积不改变空间大小，但可以**改变通道数**。它相当于对每个位置的所有通道做一个"加权汇总"。在这里，它把主干网络提取的丰富特征（512 通道），压缩成检测需要的简单输出（12 通道/3 通道）。

### 5.3 模型输入输出

| 项目 | 值 |
|------|-----|
| 模型名称 | FD_LPD_Redactor (NVIDIA) |
| 框架 | Caffe → ONNX → TensorRT |
| 输入尺寸 | 3 × 270 × 480 (CHW) |
| 输出层 | `output_bbox` (16通道), `output_cov` (4通道) |
| 网格分辨率 | 17 × 30 (每个网格对应 16×16 像素) |
| 推理精度 | FP16 (半精度浮点) |
| 构建方式 | `trtexec --onnx=fd_lpd.onnx --fp16` |

### 5.4 4 个类别

从 [labels.txt](fd_lpd_model/labels.txt) 可知：

| classId | 含义 | 来自分支 | cov 通道 | bbox 通道 |
|---------|------|----------|----------|-----------|
| 0 | face (人脸) | Branch 2 | cov[0] | bbox[0-3] |
| 1 | license_plate (车牌) | Branch 1 | cov[1] | bbox[4-7] |
| 2 | make (车辆品牌) | Branch 1 | cov[2] | bbox[8-11] |
| 3 | model (车辆型号) | Branch 1 | cov[3] | bbox[12-15] |

### 5.5 为什么人脸和车牌要做成两个独立分支？

人脸和车牌的**视觉特征差异很大**：
- 人脸：皮肤纹理、五官分布、肤色
- 车牌：高对比度边框、文字笔画边缘

用一个统一的网络去同时检测这两类，特征会互相干扰。NVIDIA 的做法是两个独立主干各自提取特征，只是共享输入图片，最后把结果拼在一起。这样每个网络可以专注自己的任务。

## 六、数据端到端流程


## 七、代码逐段解析（结合前面的知识）

### 7.1 寻找输出层（第 117-146 行）

模型的 Inference 输出可能不止两层（取决于 TensorRT 引擎构建时暴露了哪些层），所以代码需要从输出列表中找到目标层：

```cpp
// 方式1: 按名字精确匹配
find_layer_exact({"conv2d_bbox",        "output_bbox"})     → bbox 层
find_layer_exact({"conv2d_cov/Sigmoid", "output_cov"})      → cov 层

// 方式2: 如果名字匹配失败，按通道数推测
// bbox 层的通道数 = 类别数 × 4   (每个类 4 个坐标值)
// cov 层的通道数  = 类别数       (每个类 1 个置信度)
find_layer_by_dims(channels = numClassesConfigured × 4, "bbox")
find_layer_by_dims(channels = numClassesConfigured,      "cov")
```

如果两个都找不到 → 报错退出，打印所有可用层供调试。

### 7.2 遍历每个网格点（第 198-251 行）

```
for c = 0..3:        // 4 个类别
  for h = 0..16:     // 17 行网格
    for w = 0..29:   // 30 列网格
      │
      ├─ 1. 读置信度: cov_buf[c * grid_size + row*30 + col]
      ├─ 2. 低于阈值? → 跳过 (这个位置肯定没有这个类)
      ├─ 3. 读偏移量: bbox_buf[c*4*grid_size + 0/1/2/3*grid_size + i]
      ├─ 4. 解码坐标: gc_center ± offset × 35.0
      ├─ 5. 裁剪到图片范围内 (防止框跑到画面外)
      ├─ 6. 检查有效性 (right > left 且 bottom > top)
      └─ 7. 存入 objectList
```

### 7.3 bbox 数据的内存布局

这是理解代码的一个难点。模型的 bbox 输出是一个**扁平化的一维数组**：

```
bbox 张量形状: [16, 17, 30]
在内存中: float bbox_buf[16 × 17 × 30] = float bbox_buf[8160]

每个类别的 4 个坐标值 (x1, y1, x2, y2) 是连续存储的:

类别 c 的 x1 起始位置: c × 4 × grid_size + 0 × grid_size
类别 c 的 y1 起始位置: c × 4 × grid_size + 1 × grid_size
类别 c 的 x2 起始位置: c × 4 × grid_size + 2 × grid_size
类别 c 的 y2 起始位置: c × 4 × grid_size + 3 × grid_size

其中 grid_size = 17 × 30 = 510

代码中的对应:
output_x1 = output_bbox_buf + (c * 4 * 17 * 30)          // 类别 c 的 x1 开头
output_y1 = output_x1 + 510                               // 类别 c 的 y1 开头
output_x2 = output_y1 + 510                               // 类别 c 的 x2 开头
output_y2 = output_x2 + 510                               // 类别 c 的 y2 开头
```

## 九、常见疑问

### Q1: bbox_norm = 35.0 哪来的？Codex 编的吗？

**不是编的。** 35.0 是 NVIDIA DetectNet 框架的标准归一化常数。这个模型是 NVIDIA 官方 2018 年的 `FD_LPD_Redactor`，训练时就用 35.0。`fd_lpd.prototxt` 里有 NVIDIA 版权声明。

### Q2: 如果我想换自己训练的模型，需要改什么？

| 改动项 | 涉及文件 |
|--------|----------|
| 换 ONNX 模型 | [build_engine.sh](build_engine.sh) 中的 `ONNX_FILE` |
| 类别数变了 | DeepStream 配置文件中的 `num-detected-classes` |
| 标签变了 | [labels.txt](fd_lpd_model/labels.txt) |
| bbox_norm 变了 | [nvdsinfer_custom_fd_lpd.cpp:167-168](nvdsinfer_custom_fd_lpd.cpp#L167) 中的常量 |
| 输出层名变了 | [nvdsinfer_custom_fd_lpd.cpp:117-125](nvdsinfer_custom_fd_lpd.cpp#L117) 中的名称列表 |
| 网格大小变了 | 不需要改——代码自动从 `cov_dims.w/h` 读取 |

### Q3: 这个解析器有 NMS（非极大值抑制）吗？

**没有。** 代码只做了置信度过滤，没有去掉重叠框。NMS 一般在 DeepStream 的 nvinfer 插件内部完成，或者依赖这个解析器的输出格式让 DeepStream 自动处理。

### Q4: 为什么 .engine 文件不能跨 GPU/Jetson 使用？

TensorRT 引擎是为**特定的 GPU 型号 + CUDA 版本 + TensorRT 版本**编译优化的二进制文件。换设备就必须重新构建。这也是 `build_engine.sh` 存在的意义——在新设备上跑一遍就能生成针对该设备的引擎。

## 十、总结

> **一句话**：神经网络把图片分成 30×17 的网格，每个格子输出"有没有东西"（cov 置信度）和"框的精确偏移量"（bbox 坐标）。这个文件遍历所有格子，用 NVIDIA DetectNet 的解码公式（bbox_norm=35.0）把偏移量还原成真实像素坐标，裁剪后交给 DeepStream 画框。

> **知识链**：卷积提取特征 → stride 缩小图片形成网格 → ResNet 跳连保证深层网络可训练 → 1×1 卷积把特征压缩成检测输出 → Sigmoid 把置信度转成概率 → 网格中心 + 偏移量解码为像素坐标 → FP16 让模型在 Jetson 上跑得动。
